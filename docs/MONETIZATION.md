# MinuteWave Monetization Design

## Recommended Model

Use a freemium model with:

- Free tier for local-first core value
- Pro subscription for unlimited and advanced usage
- 7-day free trial for Pro

Recommended pricing starting point:

- Pro Monthly: EUR 9.99
- Pro Yearly: EUR 79.99

This can be adjusted later, but the structure should be decided now.

## Guiding Principle

Do not put basic trust or security behind a paywall.

That means encryption and basic privacy protections should remain part of the base product, not a Pro-only feature.

## Free Tier

The free tier should prove the product value quickly.

Included in Free:

- local recording and local session storage
- local transcription
- transcript viewing
- basic summaries
- limited session history
- limited monthly usage

Recommended Free limits:

- 10 sessions per month
- no cloud AI providers
- no advanced summary/chat features
- no priority export features

Goal of Free:

- let users understand the product
- demonstrate the local-first value
- create a clear reason to upgrade without making the app feel crippled

## Pro Tier

Pro is for serious recurring use.

Included in Pro:

- unlimited sessions
- unlimited local transcription usage
- cloud provider integrations
- advanced structured summaries
- chat with past sessions
- premium export/share options
- faster access to new AI features and provider support

## Features to Gate

Good Pro gates:

- unlimited session volume
- cloud AI integrations
- advanced summary templates
- session chat and retrieval features
- premium exports
- future team or collaboration features

Features not recommended to gate:

- encryption
- basic local recording
- basic transcript access
- core trust/privacy behavior

## Trial Strategy

Use a 7-day free trial for Pro.

Reason:

- users need to record at least a few sessions to feel the value
- chat, summaries, and provider integrations are easier to sell after real use than at install time

## Upgrade Triggers

Best moments to show the paywall:

- after the user has completed a few sessions
- when they try to enable a cloud provider
- when they hit the monthly free-session limit
- when they try to use advanced summary/chat features

Do not show the paywall too early in onboarding.

The app should first prove the local-first core experience.

## Positioning of Pro

Suggested Pro framing:

- Unlimited meeting capture
- More powerful summaries
- Chat with your sessions
- Bring your preferred AI provider
- Keep the same local-first workflow, with more power

## Future Expansion

Possible later tiers:

- Pro Plus: heavier AI usage or bundled cloud credits
- Team: shared settings, collaboration, admin controls, or organization billing

Do not add these yet. Start with Free + Pro only.

## Product Logic Summary

### Free

- local-first core
- useful on day one
- limited volume
- no cloud AI

### Pro

- unlimited usage
- advanced AI workflows
- provider flexibility
- premium outputs

## Final Recommendation

Ship with:

- Free
- Pro Monthly
- Pro Yearly
- 7-day Pro trial

And use this rule:

Free proves the local-first promise.  
Pro unlocks scale, power, and flexibility.
